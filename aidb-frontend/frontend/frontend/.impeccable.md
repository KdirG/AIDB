## Design Context

### Users
Database administrators, data analysts, and non-technical stakeholders who need to interact with and query SQL databases using natural language. They require speed, clarity, analytical depth, and trust in the AI's operations.

### Brand Personality
Technical, precise, futuristic, yet trustworthy. Deep obsidian themes (dark mode) with cyber emerald accents representing "success" and "go" states. "AIDB Intelligence".

### Aesthetic Direction
Modern, glassmorphic dark theme. High contrast with a focus on typography and data visibility. It should look like a command center or terminal of the future, without being cliché. Clean, modular layout with subtle animations. 

### Design Principles
1. **Clarity over Clutter:** Complex SQL queries and tabular data results must be immediately scannable.
2. **Confident Interactions:** Destructive or modifying queries must clearly denote risk (e.g., using stark color contrasts like amber/red).
3. **Fluidity:** The transition from natural language prompt to executing query to visualizing data should feel seamless and instantaneous.
